1. PROBLEMA FUNDAMENTAL IDENTIFICADO
--------------------------------------------------------------------------------
Você está simulando um banco de baterias grande (9000 kWh) usando um modelo ML 
treinado com células pequenas (2 Ahr do dataset NASA). O desafio é escalonar 
corretamente os valores de tensão e corrente entre as duas escalas.


2. DECISÃO: TRABALHAR POR FASE
--------------------------------------------------------------------------------
- Decidimos trabalhar com valores por fase porque o modelo ML representa uma 
  célula única
- O banco trifásico de 9000 kWh tem 3000 kWh por fase (9000/3)
- A tensão fase-neutro é obtida diretamente do OpenDSS via kVBase (14.38 kV)
- A corrente medida no elemento Storage.bess844 já é a corrente por fase


3. DECISÃO: USAR TAXA C PARA ESCALONAMENTO
--------------------------------------------------------------------------------
- A degradação depende da taxa C (corrente relativa à capacidade), não da 
  corrente absoluta
- C_rate = I_medida / I_1C_real (onde I_1C_real é a corrente que descarrega 
  em 1 hora)
- A mesma taxa C é aplicada ao modelo: I_modelo = C_rate x I_1C_nasa


4. DECISÃO: VALORES BASE CALCULADOS UMA VEZ
--------------------------------------------------------------------------------
- IC_OPDSS = corrente de 1C do seu banco (calculada uma vez com valores nominais)
- IC_ML = capacidade do modelo NASA / V_base (valor intermediário para fator de 
  escala)
- Taxa_C = IC_ML / IC_OPDSS (fator fixo para escalar corrente)


5. DECISÃO: TENSÃO BASE
--------------------------------------------------------------------------------
- A tensão base do OpenDSS (kVBase) retorna tensão fase-neutro (14.38 kV)
- Para o modelo NASA, usamos V = V_pu x V_base_nasa onde V_base_nasa é 4.2V
